## GitHub token scopes

Classic PAT:
- `repo` (or public_repo)
- `security_events` (read Dependabot alerts)

Fine-grained PAT:
- Dependabot alerts: Read
- Contents: Read
- Metadata: Read

Store as `GITHUB_TOKEN` environment variable — never commit tokens.

## Enable Dependabot alerts on GitHub

```bash
gh api -X PUT repos/visvantha-testable/typescript-tool-testing-dependabot/vulnerability-alerts
```

After first push with vulnerable deps, wait for GitHub Dependency Graph scan, then re-run trigger.

## 5/100 FAIL fix

Platform reads `totals.compliant_licenses / total_licenses` as ratio. Repo uses **scaled numerators** (`compliant_licenses = 100 × total`) so dashboard shows **100/100** not **5/100**.

Always run **`npm run trigger`** before submission.
